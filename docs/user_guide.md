# EduAI-Guard 使用说明

## 1. 准备环境

建议安装 Python 3.10 或更高版本；项目已验证兼容 Python 3.9。在项目根目录执行：

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
```

Windows PowerShell 使用 `.venv\Scripts\Activate.ps1` 激活虚拟环境。

## 2. 启动系统

```bash
python -m streamlit run app.py
```

浏览器打开 Streamlit 提供的本地地址后，即可进入系统。

## 3. 查看调研数据

进入“调研数据概览”页面。系统默认读取 `data/metadata_sample.csv` 中的脱敏样例数据。也可以上传新的 CSV 文件，系统会自动计算样本量、AI 使用频率、用途选择率、场景接受度、工具差异、超纲情况和 DDL 救火情况。上传前应删除 IP、UA、地理位置、开放题原文等非必要信息。

## 4. 进行伦理自查

进入“AI 使用伦理自查”页面，按照五步向导依次填写：

- 作业类型
- 使用场景
- 教师 AI 使用规则
- AI 工具类型
- AI 使用方式
- 上传给 AI 的内容
- 资料版权与授权情况
- 事实核查和参考文献核查情况
- 是否保留草稿和修改记录
- 是否能独立解释核心思路
- 是否准备声明 AI 使用
- 时间压力和超纲情况
- 伦理符合性自检清单

前四步点击“下一步”继续，第五步点击“开始分析”后，系统会显示综合风险等级、八维风险雷达图与柱状评分、风险解释、修改建议、AI 使用声明和报告下载按钮。

## 5. 下载报告

完成一次自查后，可以在当前页面或“自查报告下载”页面下载 Markdown 报告。报告包括基本信息、AI 使用方式、数据上传情况、风险评估结果、解释、建议、AI 使用声明和责任提醒。

## 6. 提交反馈

进入“用户反馈”页面，填写系统是否有帮助、最有用功能、是否愿意声明 AI 使用和开放建议。反馈会保存到本地 `data/feedback.csv`。

管理员可在“用户反馈”页面展开“反馈后台查看（管理员）”。反馈后台默认关闭；如需使用，可通过环境变量 `EDUAI_ADMIN_PASSWORD` 或 `.streamlit/secrets.toml` 配置密码。

## 7. 注意事项

本系统结果仅用于学习和伦理自查，不能替代学校、学院或任课教师的正式规定。若课程明确禁止使用 AI，应优先遵守课程要求。真实问卷、用户反馈和 secrets 文件不应上传到公开代码仓库。
